Monitool REST API Documentation

/data/

Collection of data aggregated from Sensors. Contains cpu and memory load.

GET

Use to get all data available

URI

/data/

Example response:

{
    "data": [
        {
            "itemID": 123,
            "sensorToken": "TOKEN",
            "date": "2015-03-01 00:00:00",
            "cpuLoad": 1.23,
            "memLoad": 0.123
        },
        ...
    ]
}

GET - queries

Use to get specified data

Parameter

/data/?sensorToken={TOKEN}

Example Query

/data/?sensorToken=098f6bcd4621d373cade4e832627b4f6

Parameter

/data/?date={START},{END}
/data/?date={START},
/data/?date=,{END}

Example Query

/data/?date="2015-03-01 00:00:00","2015-03-01 12:00:00"

POST

Insert new data from sensor

URI

/data/

Example query:

{
    "sensorToken": "098f6bcd4621d373cade4e832627b4f6",
    "data": {
        "cpuLoad": 1.23,
        "memLoad": 0.123
    }
}

Example response:

{
    "itemID": 123
}

/complexStat/

Collection of ongoing Complex Statistics.

Field Available Values Comment
statType AVG - average,
MAX - maximum,
MIN - minimum,
MED - median		 Defines kind of statistics
itemType CPU - load of CPU,
MEM - load of RAM		 Defines type of data
period seconds Defines duration in which the statistic will be computed on each iteration
repeat seconds Defines how often the iteration will occur

GET

URI

/complexStat/

Example response:

{
    "stats": [
        {
            "statID": 123,
            "statType": "AVG",
            "itemType": "CPU/MEM",
            "period": 300,
            "repeat": 60
        },
        ...
    ]
}

POST

Insert new complex statistics

URI

/complexStat/

Example query:

{
    "stat": {
        "statType": "AVG",
        "itemType": "CPU/MEM",
        "period": 300,
        "repeat": 60
    }
}

Example response:

{
    "statID": 123
}
